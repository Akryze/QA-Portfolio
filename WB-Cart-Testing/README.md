# Демонстрация тестирования модуля «Корзина» Wildberries 

Проект по функциональному и регрессионному тестированию критического пути пользователя. Демонстрация части навыков работы с TMS и некоторыми инструментами отладки.

---

### Стек технологий и инструменты
* **TMS:** [Test IT](https://testit.software/) — управление тест-кейсами и прогонами.
* **Network Analysis:** Chrome DevTools (анализ API-трафика).
* **Environment:** Windows 11, Google Chrome (последняя версия).

---

### Что было сделано:

1. **Test Design:** * Разработаны чек-листы и тест-кейсы (функционал добавления, UI корзины, валидация промокодов).
   * Спроектированы негативные сценарии для проверки устойчивости системы.

2. **Execution (Test Run):**
   * Проведен прогон тестов через Test IT.
   * Фиксация результатов (Passed/Failed) и сбор доказательств (скриншоты, логи).

3. **API Investigation:**
   * При проверке промокодов использовалась вкладка **Network** для отслеживания ответов от бэкенда.

### Найденный дефект (Bug Report)

> [!IMPORTANT]
> **Примечание:** Данный баг зафиксирован в рамках учебного проекта. Он служит для демонстрации навыка локализации ошибок (Server Error 500) через Chrome DevTools и их документирования в TMS.

**Summary:** [API] Статус 500 (Internal Server Error) при вводе некорректного промокода.
**Severity:** High
**Priority:** High

**Steps to Reproduce:**
1. Добавить товар в корзину.
2. В поле промокода ввести произвольное значение (например, `zxcvbn`).
3. Нажать кнопку «Применить».

**Expected Result:**
Появление валидационного сообщения «Такого промокода нет». Статус HTTP-запроса `/promocodes` — **200 OK** (с телом ошибки) или **404 Not Found**.

**Actual Result:**
Интерфейс не меняется, в Network зафиксирована ошибка **500 Internal Server Error**.
